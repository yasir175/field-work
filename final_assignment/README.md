# Product Store — Flutter Assignment

A simple Flutter application that fetches products from the **DummyJSON Products API**, caches them locally with **SQLite**, and supports full CRUD, search, pagination, offline access, and pull-to-refresh.

This project was built as a beginner-level university assignment. The code intentionally uses simple `StatefulWidget` + `setState()` state management, basic Material widgets, and a flat folder structure — no Clean Architecture, no Bloc/Riverpod/GetX, no Firebase.

## API Used

**DummyJSON Products API** — https://dummyjson.com/products

- `GET /products?limit=&skip=` — paginated product list
- `GET /products/search?q=` — search (available but not required, since search is done locally on already-loaded data)

DummyJSON does not persist Create/Update/Delete requests on its server, so all CRUD operations are simulated locally using SQLite, while the product list itself is still fetched live from the API.

## Packages Used

| Package | Purpose |
|---|---|
| `http` | Calling the DummyJSON REST API |
| `sqflite` | Local SQLite database (offline cache + local CRUD) |
| `path` | Building the database file path |
| `path_provider` | Locating the device's database directory |
| `cached_network_image` | Loading and caching product images |

## Features

- **Splash Screen** — logo + app name, auto-navigates to Home after 2 seconds
- **Home Screen** — product list with image, title, category, description, price
- **Search** — simple live filter by title or category
- **Detail Screen** — full product info: image, description, category, price, rating, stock, brand
- **CRUD** — Add, Edit, Delete products, all stored locally in SQLite
- **Local Cache / Offline Mode** — on launch, fetches from the API and saves to SQLite; if there's no internet, loads the last cached data instead
- **Pagination** — loads 10 products at a time with a "Load More" button
- **Error Handling** — loading spinners, SnackBars with the actual error message, and a retry button for no internet / API errors / empty list / database failures
- **Pull to Refresh** — swipe down on Home Screen to re-fetch from the API
- **Responsive Layout** — simple grid layout on wide (tablet) screens, list layout on phones

## How to Run

1. Make sure Flutter is installed: https://docs.flutter.dev/get-started/install
2. From the project root, get dependencies:
   ```
   flutter clean
   flutter pub get
   ```
3. Run on an **Android emulator/device or iOS simulator** — not Chrome/web and not Windows/macOS/Linux desktop. The `sqflite` package used here only supports Android and iOS.
   ```
   flutter run
   ```
4. To build a release APK:
   ```
   flutter build apk --release
   ```

### Troubleshooting

- **"Add Product" seems to do nothing / database not saving:** Open Logcat / the Run panel in Android Studio while adding a product. `database_helper.dart` logs every operation, e.g. `DatabaseHelper: inserted product id=...` on success or `DatabaseHelper: insertProduct FAILED -> ...` with the real error on failure. The on-screen SnackBar shows the same message.
- **Running in a browser (Chrome/Edge):** `sqflite` does not support web at all — every database call will fail. Run on an Android emulator, iOS simulator, or a physical Android device instead.
- **Android emulator won't start on Windows:** usually a virtualization conflict. Enable **Windows Hypervisor Platform** and **Hyper-V** in "Turn Windows features on or off", install the **Android Emulator Hypervisor Driver (AEHD)** from Android Studio's SDK Manager → SDK Tools, then restart your PC.
- **Copied `lib/` and `pubspec.yaml` into a fresh Android Studio project:** after replacing `pubspec.yaml`, always run `flutter clean && flutter pub get` and rebuild — otherwise newly added native plugins (like `sqflite`) won't be linked into the Android build.

## Project Overview

On launch, the app shows a splash screen, then tries to fetch the first 10 products from the DummyJSON API. Successfully fetched products are saved to a local SQLite database. If the device has no internet connection (or the initial fetch fails), the app automatically falls back to showing whatever products were cached from the last successful fetch, and shows a clear error message if neither the API nor the cache has any data.

From the Home Screen, users can search products, tap "Load More" to fetch additional pages, pull down to refresh, and tap a product to view its full details. The Floating Action Button and each product's Edit/Delete buttons open a shared Add/Edit form, allowing users to fully manage their own local product list on top of the data fetched from the API. Every database operation is wrapped in error handling so failures are surfaced to the user via a SnackBar and logged to the console, instead of failing silently.
